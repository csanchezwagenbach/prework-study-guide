HTML

  • In HTML, an 'element' refers to a type of content on a webpage.
  • Some element types used to add content to a webpage are text elements, image elements, header elements, and list elements.
  • Common element types used to divide and describe a page include meta-related elements, content division elements, and semantic elements.
  • An HTML attribute defines additional information about an element and is always defined in the beginning element, never the closing element.
  • The head element contains information about the webpage pertinent to the web browser, often called metadata.
  • Metadata tags found in the head element include the title (defines title for webpage, a required element and critical for Search Engine Optimization [SEO]), style (defines CSS styles associated with webpage), base (defines the base URL for a webpage), link (connects external resources to HTML doc), meta (character set, description, keywords, author, and viewport), and script elements (embeds client-side scripts in HTML doc).
  • External CSS style sheets are linked after the final meta element and before the title element.
  • Links follow the uniform format link rel="stylesheet" href="relative file location".
  • rel= determines the relationship between the current HTML file and the linked document resource.
  • href= describes the location of the stylesheet relative to the current HTML file.
  • The body element represents the visible content shown to the user.
  • Elements nested within other elements are referred to as child elements. Reciprocally, the element that holds the children is the parent element.
  • There are a number of online resources to assist with identifying and deploying the diverse organizational tools within the body element of an HTML file!

CSS

  • There are three methods of adding CSS to an HTML file: inline CSS, interal style sheets, and external CSS style sheets.
  • Inline CSS applies only to a specific HTML tag and only styles that particular page element.
  • An internal style sheet contains CSS rules for the webpage in the head section of an HTML file. The rules only apply to that given page.
  • External CSS style sheets are the preferred method. External style sheets allow all CSS rules to be kept in a separate file, facilitating design changes and adhering to the Separation of Concerns principle.
  • When using an external CSS style sheet, remember that it must be linked in the HTML file's head section.
  • External style sheets are linked after the final meta element and before the title element.
  • CSS Rules have two components, a selector and a declaration block.
  • The selector defines the element or attributes to which the ruels, or declarations, will apply.
  • Declarations contain two important components, the CSS property one wants to apply and the value of that property.
  • Declarations follow the uniform format "property: value;".
  • Class selectors allow a CSS specification to be applied to all given HTML elements assigned to that class. Class selectors are preceded by a period in this manner, .given-class
  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.

Git

  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it
  • Git Flow refers to the regular process of syncing the local and remote repositories.
  • First, changed files in your working directory must be sent to the staging area of Git. Use the command 'git add -A' to stage all changes made.
  • After staging changes, they can be commit to the remote repository with a message using the command git commit -m "message describing changes made typed here".
  • Before opening a pull request, use command 'git pull origin main' to ensure current branch is up to date.
  • Finally, use command 'git push origin branch-name' to open pull request on remote repository.

JavaScript

  • A variable is a named container that allows us to store data in our code.
  • Control flow is the order in which a computer executes code in a script.
  • A function is a defined set of instructions that can be called and executed.
  • Remember, after defining a function, it must be called in order to execute itself!
  • Multiple pieces of data may be stored in a variable. The collection of data is known as an array.
  • Conditional statements may be used to intervene in the control flow and determine what code is to be executed.
  • Loops may be used to execute a given task repeatedly. Three statements define a loop.
  • First, the starting poing for the loop. To start at the beginning of an array, declare a variable x and give it a value of 0.
  • Next is the condition. The condition determines the number of times the loop will be kept running. To run a loop for the duration of an array, set the variable x to be less than the 'array.length'.
  • The final statement determines how the array will iterate the items. To iterate each item in an array, use the code x++, which is a shortcut for telling x to increase by 1 for each run of the loop
  • Like a CSS stylesheet, a .js file must be linked to the HTML file.
